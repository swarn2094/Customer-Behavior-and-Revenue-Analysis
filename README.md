# Customer-Behavior-and-Revenue-Analysis

## Overview
This is an end-to-end **Data Analytics Project** that analyzes customer purchasing behavior using **Python, PostgreSQL, and Power BI**. The project demonstrates the complete analytics workflow, including data loading, cleaning, exploratory data analysis (EDA), SQL-based data querying, and the development of an interactive Power BI dashboard. The objective is to uncover customer insights, identify purchasing trends, and support data-driven business decisions.

---

## Dataset
The dataset contains customer shopping and transaction information, including:

- Customer Demographics (Age, Gender)
- Product Categories
- Purchase Amount
- Subscription Status
- Shipping Type
- Review Ratings
- Sales and Revenue Information

## Project Workflow

### 1. Data Loading
- Imported the dataset into Python.
- Inspected the data structure and data types.

### 2. Data Cleaning
- Removed duplicate records.
- Handled missing values.
- Corrected data types.
- Prepared the dataset for analysis.

### 3. Exploratory Data Analysis (EDA)
- Analyzed customer demographics.
- Explored purchasing patterns.
- Examined product category performance.
- Identified trends in customer spending and ratings.

### 4. SQL Analysis (PostgreSQL)
- Imported the cleaned dataset into PostgreSQL.
- Wrote SQL queries to answer business questions.
- Generated summary statistics and category-level insights.
- Performed aggregations using GROUP BY, ORDER BY, COUNT, SUM, and AVG.

### 5. Power BI Dashboard
Built an interactive dashboard to visualize:

- Customer Overview
- Sales by Category
- Revenue by Category
- Revenue by Age Group
- Sales by Age Group
- Subscription Status Distribution
- Average Purchase Amount
- Average Review Rating

Interactive slicers include:

- Gender
- Product Category
- Subscription Status
- Shipping Type
